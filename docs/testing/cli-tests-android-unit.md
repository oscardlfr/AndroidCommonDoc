# Android Unit Tests

Android unit test execution (Robolectric, no device required) via kmp-test-runner v0.9.0.

## CLI Usage

```bash
# Android unit tests (debug variant, AGP 9 default)
kmp-test parallel --test-type androidUnit

# With coverage
kmp-test parallel --test-type androidUnit --coverage-tool auto

# Single module
kmp-test parallel --test-type androidUnit --module app:android
```

## Gradle Task Reference (BLOCKED — use CLI instead)

| Gradle Task | What It Runs | AGP Notes | Gate Status |
|-------------|-------------|-----------|-------------|
| `androidUnitTest` | All unit test variants | Aggregates all build types | BLOCKED |
| `testDebugUnitTest` | Debug variant unit tests | AGP 9 default enabled type | BLOCKED |
| `testReleaseUnitTest` | Release variant unit tests | Disabled by default in AGP 9 | BLOCKED |
| `testDebug` | Alias for testDebugUnitTest | May vary by AGP version | BLOCKED |

## AGP 9 Behavior Change

`android.onlyEnableUnitTestForTheTestedBuildType` changed default from `false` → `true` in AGP 9.0.

**Effect**: `testReleaseUnitTest` is **no longer generated by default**. Only `testDebugUnitTest` is active unless explicitly opted in.

```kotlin
// build.gradle.kts — opt-in for release tests (AGP 9+)
android {
    testOptions {
        unitTests.isIncludeAndroidResources = true
    }
    // To restore release unit tests:
    onlyEnableUnitTestForTheTestedBuildType = false
}
```

## Robolectric Notes

- Robolectric tests run on JVM without a device
- Use `testImplementation("org.robolectric:robolectric:...")` scoped to `androidUnitTest`
- AGP 9 may require `@Config(manifest = Config.NONE)` for modules without a manifest
