# Requirements: AndroidCommonDoc

**Defined:** 2026-03-12
**Core Value:** Patterns and skills must be accurate, current, and token-efficient — if a developer follows these patterns and uses these skills, their code is correct by construction and their AI agent spends tokens on logic, not boilerplate.

## v1 Requirements

### Pattern Documentation

- [x] **PTRN-01**: All KMP architecture layers have complete pattern documentation (ViewModel, UI, testing, Gradle, offline-first, resources, compose resources)
- [x] **PTRN-02**: Code samples in pattern docs are compilable and verified against current library versions
- [x] **PTRN-03**: Automated freshness tracking detects stale version references and deprecated API usage in docs

### Cross-Platform Scripts

- [x] **SCRP-01**: All script parameters use consistent naming across PS1, SH, Claude commands, and Copilot prompts (fix --project-path vs --project-root drift)
- [x] **SCRP-02**: Script parameter manifest exists as single source of truth for all script interfaces
- [x] **SCRP-03**: Automated parity test suite verifies PS1 and SH scripts produce identical behavior for same inputs

### Build-Time Enforcement

- [x] **LINT-01**: Custom Detekt rule set enforces top architecture patterns (sealed UiState, CancellationException rethrow, no platform deps in ViewModels, WhileSubscribed timeout, no Channel for UI events)
- [x] **LINT-02**: Convention plugins in build-logic/ enable one-line Gradle adoption of all enforcement rules in consuming projects
- [x] **LINT-03**: Compose Rules (mrmans0n 0.5.6) integrated for Compose-specific lint enforcement

### Multi-AI-Tool Support

- [x] **TOOL-01**: Canonical skill definition format generates tool-specific files for Claude Code, GitHub Copilot, and future tools
- [x] **TOOL-02**: AGENTS.md universal instruction format adopted as base layer for all AI tool integrations
- [x] **TOOL-03**: Claude Code hooks enforce patterns in real-time during AI-assisted development

### Quality Infrastructure

- [x] **QUAL-01**: Quality gate agents validate internal consistency (script parity, skill-script alignment, template sync, doc-code drift)
- [x] **QUAL-02**: Automated cross-surface parameter drift detection catches mismatches across Claude/Copilot/scripts before release
- [x] **QUAL-03**: Token cost measurement per skill validates efficiency claims with actual data

## v2 Requirements

### Architecture Testing

- **ARCH-01**: Konsist architecture tests enforce structural rules as unit tests
- **ARCH-02**: Architecture guard tests for consuming projects (dependency direction, layer violations)

### Extended AI Tool Support

- **XTOL-03**: MCP server exposes toolkit capabilities as tool endpoints

### iOS/SwiftUI Patterns

- **IOS-01**: SwiftUI navigation patterns documentation

- **IOS-03**: appleMain/iosMain/macosMain source set guidance

## Out of Scope

| Feature | Reason |
|---------|--------|
| Runtime libraries or compiled SDKs | Toolkit is docs + scripts + skills, not compiled code that apps depend on |
| Business logic from DawSync/WakeTheCave/OmniTrack | Generic toolkit — no app-specific patterns |
| CI/CD pipeline configurations | Provide lint rules and scripts, not pipeline definitions — too org-specific |
| IDE plugins (Android Studio, IntelliJ) | High maintenance burden, Detekt/ktlint already integrate with IDEs |
| Full project templates/scaffolding | Anti-pattern per research — forces structure, kills incremental adoption |
| Detekt 2.0 rules | Target 1.23.8 stable; migrate when Kotlin 2.3+ is mainstream in consuming projects |
