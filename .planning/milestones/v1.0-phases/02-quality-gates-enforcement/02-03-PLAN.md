**script-parity-validator.md implementation body:**

Instruct the agent to perform static analysis (NOT execution) of script pairs:

1. **PAIRING:** Glob `scripts/ps1/*.ps1` and `scripts/sh/*.sh`. Match pairs by base name (e.g., `gradle-run.ps1` <-> `gradle-run.sh`). Report `[MISSING]` for unpaired scripts, `[OK]` for matched pairs. Also check `scripts/ps1/lib/` vs `scripts/sh/lib/` for utility script pairing.

2. **FLAGS:** For each paired script:
   - PS1: Read the file, find `param()` block, extract parameter names (e.g., `$ProjectRoot`, `$Module`, `$Retry`). Convert PascalCase to kebab-case for comparison.
   - SH: Read the file, find `getopts` or `while ... case` blocks, extract flag names (e.g., `--project-root`, `--module`, `--retry`).
   - Cross-reference against `skills/params.json` which maps canonical parameter names to both PS1 and SH forms.
   - Report `[MISMATCH]` for flags present in one but not the other, `[OK]` for matched.

3. **OUTPUT FORMAT:** For each pair, compare output patterns:
   - Look for echo/Write-Host statements that produce structured output (tables, headers, separators)
   - Compare format strings and output structure
   - Report `[DRIFT]` for format differences, `[OK]` for matching formats

4. **EXIT CODES:** For each pair:
   - PS1: Find `exit` statements, note exit code values and conditions
   - SH: Find `exit` statements, note exit code values and conditions
   - Report `[MISMATCH]` for differing exit code semantics, `[OK]` for matching

5. **LIBRARIES:** Check `scripts/ps1/lib/` vs `scripts/sh/lib/` and `scripts/lib/` (shared):
   - Flag utilities in one platform but not the other
   - Confirm shared utilities (Python in scripts/lib/) are used by both

6. **Output:** Produce the structured report matching the existing output format spec exactly.

**skill-script-alignment.md implementation body:**

Instruct the agent to verify command-to-script mappings:

1. **MAPPING:** Read each `.claude/commands/*.md`. Find implementation code blocks (bash/powershell fenced blocks). Extract script references (paths like `scripts/sh/gradle-run.sh`). Verify each referenced script exists. Report `[BROKEN]` for missing scripts, `[OK]` for valid references.

5. **OUTPUT:** Compare command's output description against what the script actually prints. Report `[DRIFT]` for mismatches, `[OK]` for matching.

6. **Output:** Produce the structured report matching the existing output format spec exactly.

Both agents use Read, Grep, Glob tools only (as declared in their frontmatter). No Bash execution.
