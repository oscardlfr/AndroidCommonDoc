AST detection strategies per rule (from research):
- **SealedUiStateRule:** `visitClass()` -- check classes/interfaces named `*UiState` for `sealed` modifier
- **CancellationExceptionRethrowRule:** `visitCatchSection()` -- check catch clauses for CancellationException (and bare Exception/Throwable) without rethrow. Walk catch body for throw expressions.
- **NoPlatformDepsInViewModelRule:** Two-pass: (1) `visitClass()` to detect `: ViewModel()` inheritance, (2) `visitImportDirective()` to flag `android.*`, `java.*` (except java.io.Serializable which KMP allows), `javax.*`, `platform.*`, `UIKit` imports. Only report when BOTH conditions met in same file.
- **WhileSubscribedTimeoutRule:** `visitCallExpression()` -- find `WhileSubscribed` calls. Check for no-arg version or zero timeout arg. Accept `5000`, `5_000`, or any non-zero positive value.
- **NoChannelForUiEventsRule:** Similar two-pass to NoPlatformDeps: detect ViewModel classes, then flag `Channel` property declarations (look for `Channel<` or `Channel(` in property type/initializer).
