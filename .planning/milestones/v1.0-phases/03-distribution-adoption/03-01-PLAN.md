Purpose: Enables one-line Gradle adoption for consuming projects. Instead of manually configuring Detekt, adding the custom rules JAR, and setting up Compose Rules, consumers just apply the plugin and get everything.
Output: Complete build-logic/ module with precompiled script plugin, extension DSL for per-concern opt-out, and bundled detekt config.

Create the build-logic/ Gradle module:

1. **build-logic/settings.gradle.kts**: Set `rootProject.name = "build-logic"`. Nothing else needed -- this is the simplest possible settings file for a convention plugin build.

2. **build-logic/build.gradle.kts**: Apply `kotlin-dsl` plugin. Add repositories (mavenCentral, gradlePluginPortal). Add implementation dependency on `dev.detekt:dev.detekt.gradle.plugin:2.0.0-alpha.2` so the precompiled script plugin can reference the Detekt plugin API. This is NOT the old `io.gitlab.arturbosch.detekt` artifact -- it was migrated in Detekt 2.0.

3. **build-logic/src/main/kotlin/com/androidcommondoc/gradle/AndroidCommonDocExtension.kt**: Create the extension DSL class. Three abstract `Property<Boolean>` fields:
   - `detektRules` (default: true) -- enables/disables custom Detekt architecture rules JAR
   - `composeRules` (default: true) -- enables/disables Compose Rules 0.5.6
   - `testConfig` (default: true) -- enables/disables test configuration conventions (maxParallelForks=1 for Windows file locking)

   Use `convention(true)` in init block for defaults. Per user decision: "everything enabled by default, but consuming projects can disable individual concerns."

   DSL usage example the extension enables:
   ```kotlin
   androidCommonDoc {
       detektRules.set(false) // Disable custom architecture rules
   }
   ```

  <action>
Create the precompiled script plugin that becomes `id("androidcommondoc.toolkit")`:

**File: build-logic/src/main/kotlin/androidcommondoc.toolkit.gradle.kts**

This file's name (minus `.gradle.kts`) becomes the plugin ID. Structure:

1. **Apply Detekt plugin**: `plugins { id("dev.detekt") }` -- uses the new Detekt 2.0 plugin ID per RESEARCH.md pitfall #6.

2. **Create extension**: `val toolkitExt = extensions.create<com.androidcommondoc.gradle.AndroidCommonDocExtension>("androidCommonDoc")` -- registers the opt-out DSL.

3. **Configure Detekt base settings**:
   - `buildUponDefaultConfig = true`
   - `parallel = true`
   - `config.setFrom()` pointing to the bundled config.yml from the detekt-rules module. Use a provider that resolves the AndroidCommonDoc root relative to the build-logic directory. The convention plugin lives in `build-logic/` which is one level inside AndroidCommonDoc, so the detekt-rules config is at `../detekt-rules/src/main/resources/config/config.yml` relative to build-logic root. Use `rootProject.layout.projectDirectory.dir("../detekt-rules/src/main/resources/config").file("config.yml")` or resolve from the settings' included build path. IMPORTANT: Since this is a composite build, the consuming project's rootProject is NOT AndroidCommonDoc. Instead, use a property or environment variable approach. The safest way: set `config.setFrom()` using `file()` with the `ANDROID_COMMON_DOC` env var: `file("${System.getenv("ANDROID_COMMON_DOC") ?: error("Set ANDROID_COMMON_DOC env var")}/detekt-rules/src/main/resources/config/config.yml")`. Add a fallback that also checks for the config relative to the plugin's own classpath resources.

4. **Add detektPlugins dependencies conditionally** using `afterEvaluate` to respect the extension opt-out values:
   - If `toolkitExt.detektRules.get()` is true: add `detektPlugins(files("${System.getenv("ANDROID_COMMON_DOC")}/detekt-rules/build/libs/detekt-rules-1.0.0.jar"))` -- the pre-built custom rules JAR.
   - If `toolkitExt.composeRules.get()` is true: add `detektPlugins("io.nlopez.compose.rules:detekt:0.5.6")`.

5. **Configure test conventions** if `toolkitExt.testConfig.get()` is true:
   - Set `tasks.withType<Test> { maxParallelForks = 1 }` -- per project CLAUDE.md: "Sequential execution: maxParallelForks = 1 (file locking on Windows)".
   - Set `useJUnitPlatform()`.

6. **Add a helpful error message** if ANDROID_COMMON_DOC env var is not set -- fail with clear instructions rather than cryptic NullPointerException.

IMPORTANT per RESEARCH.md pitfall #1: detektPlugins dependencies must be added in the SAME project where `dev.detekt` is applied, which this plugin script does correctly. Per pitfall #2: document in a comment that after rebuilding detekt-rules JAR, Gradle daemon may need `--stop`.

Verify the plugin compiles: `cd build-logic && ./gradlew assemble` (or use the parent Gradle wrapper if no local wrapper). If build-logic has no Gradle wrapper, that is fine -- it will be built via composite build from the consuming project. But we need to verify it compiles, so copy the Gradle wrapper from detekt-rules/ (same Gradle 9.1.0) into build-logic/ to enable standalone verification.
