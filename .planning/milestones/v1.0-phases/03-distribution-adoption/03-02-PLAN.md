**1. .claude/hooks/detekt-post-write.sh** (PostToolUse on Write|Edit):

```
#!/bin/bash
set -euo pipefail
```

- Read JSON from stdin into INPUT variable using `cat /dev/stdin`
- Extract FILE_PATH using `jq -r '.tool_input.file_path'` -- CRITICAL: all debug output to stderr (`>&2`), only JSON to stdout per RESEARCH.md pitfall #3
- Early exit (exit 0) if file does not end in `.kt` -- non-Kotlin files are allowed
- Determine ANDROID_COMMON_DOC location: check env var first, then fall back to resolving from script's own location (`SCRIPT_DIR/../..` since hook is in `.claude/hooks/` inside AndroidCommonDoc)
- Verify RULES_JAR exists at `$COMMON_DOC/detekt-rules/build/libs/detekt-rules-1.0.0.jar` -- if not, exit 0 silently (JAR not built yet, don't block development)
- Verify CONFIG exists at `$COMMON_DOC/detekt-rules/src/main/resources/config/config.yml`
- Run Detekt CLI: `java -jar "$DETEKT_CLI_JAR" --input "$FILE_PATH" --config "$CONFIG" --plugins "$RULES_JAR" 2>&1`
  - For the Detekt CLI JAR: the custom rules JAR alone is NOT the CLI. The hook needs detekt-cli.jar. Resolution approach: check for `$COMMON_DOC/detekt-rules/build/libs/detekt-cli-2.0.0-alpha.2-all.jar` first. If not present, try to download it to `$COMMON_DOC/.cache/detekt-cli-2.0.0-alpha.2-all.jar` from Maven Central (`https://repo1.maven.org/maven2/dev/detekt/detekt-cli/2.0.0-alpha.2/detekt-cli-2.0.0-alpha.2-all.jar`). Cache the download so subsequent invocations are fast. If download fails (no internet), exit 0 silently -- don't block development.
  - Run: `java -jar "$DETEKT_CLI_JAR" --input "$FILE_PATH" --config "$CONFIG" --plugins "$RULES_JAR" 2>&1`
- If Detekt exits non-zero AND output contains findings, emit blocking JSON:
  ```json
  {"decision": "block", "reason": "AndroidCommonDoc pattern violations found. Fix these before proceeding:\n<violations>"}
  ```
  Use `jq -n` to construct valid JSON with proper escaping.
- If Detekt passes (exit 0 or no findings), exit 0 with no output (allow).
- Set timeout-friendly: skip files >500 lines (`wc -l`) to avoid hook timeout per RESEARCH.md pitfall #4.

**2. .claude/hooks/detekt-pre-commit.sh** (PreToolUse on Bash):

```
#!/bin/bash
set -euo pipefail
```

- Read JSON from stdin, extract COMMAND using `jq -r '.tool_input.command'`
- Early exit (exit 0) if COMMAND does not match `git commit` pattern -- only intercept commits
- Get staged Kotlin files: `git diff --cached --name-only --diff-filter=ACM | grep '\.kt$' || true`
- Early exit if no staged .kt files
- Resolve ANDROID_COMMON_DOC (same logic as post-write)
- Resolve Detekt CLI JAR (same caching logic as post-write -- extract the shared resolution into a function or keep inline since both scripts are self-contained)
- Read ANDROID_COMMON_DOC_MODE env var (default: "block") for configurable severity per user decision
- Run Detekt on comma-separated staged file paths: `--input "file1.kt,file2.kt"`
- If violations found:
  - In block mode: emit `hookSpecificOutput` with `permissionDecision: "deny"` and the violations as reason
  - In warn mode: emit `hookSpecificOutput` with `permissionDecision: "allow"` and violations as `additionalContext`
- If clean: exit 0 silently

Both scripts must:
- Use `set -euo pipefail`
- Send ALL non-JSON output to stderr
- Produce valid JSON or no output (never mixed text+JSON)
- Handle missing dependencies gracefully (exit 0, don't block)
- Be executable (`chmod +x`)

Verify hook scripts produce valid JSON output for test inputs:
```bash

```
