<objective>
Fix the testConfig opt-out timing bug in the convention plugin.

Purpose: The testConfig extension property is read at plugin application time (line 69), before the consuming project's `androidCommonDoc { }` DSL block executes. This means `testConfig.set(false)` has no effect -- the property always reads its default (`true`). The `detektRules` and `composeRules` properties are correctly guarded inside `afterEvaluate`, but `testConfig` was left outside. This gap closure wraps the testConfig block in `afterEvaluate` to match the established pattern.
