**adapters/claude-adapter.sh**: Bash script that reads all `skills/*/SKILL.md` files and `skills/params.json`, generates `.claude/commands/*.md` files.

The adapter must:
1. Find all `skills/*/SKILL.md` files
2. For each SKILL.md, parse:
   - Frontmatter: name, description
   - Sections: Usage Examples, Parameters, Behavior, Implementation (macOS/Linux and Windows blocks), Expected Output
3. Read `skills/params.json` to get parameter definitions for the Arguments section
4. Generate a Claude command file at `.claude/commands/<name>.md` with this structure:
   ```
   <!-- GENERATED from skills/<name>/SKILL.md -- DO NOT EDIT MANUALLY -->
   <!-- Regenerate: bash adapters/generate-all.sh -->
   # /<name> - <description>

   <description expanded>

   ## Usage

   ```
   /<name> <usage patterns from params>
   ```

   ## Arguments

   <parameter list with types and defaults, derived from params.json entries listed in SKILL.md frontmatter>

   ## Examples

   <from Usage Examples section>

   ## Behavior

   <from Behavior section>

   ## Implementation

   ### macOS / Linux
   ```bash
   <from Implementation > macOS / Linux section>
   ```

   ### Windows
   ```powershell
   <from Implementation > Windows section>
   ```

   ## Output

   <from Expected Output section>
   ```

Implementation approach: Use `awk` or `sed` to parse the SKILL.md sections (they're delimited by `## ` headers). Use a simple JSON parser for params.json -- either `python3 -c "import json..."` (available on all platforms) or `node -e "..."`. Prefer `python3` since it's more universally available.

The script must be idempotent -- running it twice produces identical output.

   ---
   mode: agent
   description: "<description from SKILL.md>"
   ---

   <description>

   <Parameter prompts using ${input:name:description} syntax from params.json copilot mapping>

   ## Implementation

   ### macOS / Linux
   ```bash
   <from Implementation, with ${input:...} variable substitution for parameters>
   ```

   ### Windows (PowerShell)
   ```powershell
   <from Implementation, with ${input:...} variable substitution for parameters>
   ```
   ```

The Copilot adapter must substitute canonical parameter references with `${input:...}` syntax using the `mapping.copilot` values from params.json.

**adapters/generate-all.sh**: Simple runner that executes both adapters:

echo "Done. Generated files are in .claude/commands/ and setup/copilot-templates/"
```
