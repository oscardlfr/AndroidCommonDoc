    - Test: createDefaultManifest() generates valid include-all manifest with given l0_source path
    - Test: Checksums map accepts relative paths as keys and sha256:hex as values
    - Test: l2_specific.commands/agents/skills all default to empty arrays
    - Test: last_synced is valid ISO datetime string

  <action>
    Create `mcp-server/src/sync/` directory and `manifest-schema.ts` that:
    1. Defines Zod schema for l0-manifest.json:
       ```
       {
         version: 1 (literal),
         l0_source: string (relative path to AndroidCommonDoc),
         last_synced: string (ISO datetime),
         selection: {
           mode: "include-all" | "explicit",
           exclude_skills: string[] (default []),
           exclude_agents: string[] (default []),
           exclude_commands: string[] (default []),
           exclude_categories: string[] (default [])
         },
         checksums: Record<string, string> (path -> "sha256:hex"),
         l2_specific: {
           commands: string[] (default []),
           agents: string[] (default []),
           skills: string[] (default [])
         }
       }
       ```
    2. Export `Manifest` type inferred from Zod schema
    3. Export `validateManifest(data: unknown): Manifest` - parses and validates, throws ZodError on invalid
    4. Export `createDefaultManifest(l0Source: string): Manifest` - creates include-all manifest with empty checksums and empty l2_specific, current timestamp for last_synced
    5. Export `readManifest(filePath: string): Promise<Manifest>` - reads JSON file and validates
    6. Export `writeManifest(filePath: string, manifest: Manifest): Promise<void>` - writes with 2-space indent

    Use Zod (already a dependency in mcp-server/package.json -- verify, if not, add it). If Zod is not present, use a manual validation approach with TypeScript type guards.

    Write tests FIRST (RED), then implement (GREEN).
  </action>
