    - runScript handles script not found gracefully (returns exitCode 127 with error message)

    - RateLimiter sliding window: after oldest call expires, new calls are allowed

  <action>
1. **src/utils/script-runner.ts** -- Cross-platform script execution (Pattern 5 from RESEARCH.md):
   - Export `runScript(scriptBaseName: string, args: string[], rootDir: string, timeoutMs?: number): Promise<ScriptResult>`
   - `ScriptResult`: `{ stdout: string, stderr: string, exitCode: number }`
   - Use `execFile` from `node:child_process` (NOT `exec` -- prevents shell injection per security hook)
   - Use `promisify(execFile)` for async/await
   - Construct script path: `path.join(rootDir, "scripts", "sh", scriptBaseName + ".sh")`
   - Spawn via `execFile("bash", [scriptPath, ...args], { timeout, env: { ...process.env, NO_COLOR: "1", ANDROID_COMMON_DOC: rootDir }, cwd: rootDir })`
   - Default timeout: 30000ms
   - Catch errors: extract stdout/stderr/code from the error object
   - Export `stripAnsi(text: string): string` -- safety net regex to strip any remaining ANSI codes

2. **src/utils/rate-limiter.ts** -- Sliding window rate limiter (locked decision: configurable rate limiting):
   - Export `class RateLimiter { constructor(maxCalls: number, windowMs: number); tryAcquire(): boolean; reset(): void; }`
   - Sliding window: store timestamps of calls, prune expired, check count < max
   - Default: 30 calls per 60 seconds (generous enough for normal use, catches runaway loops)

3. **src/tools/check-freshness.ts** -- Doc freshness tool:
   - Export `registerCheckFreshnessTool(server: McpServer): void`
   - Input schema: `{ projectRoot?: string }` (optional, defaults to ANDROID_COMMON_DOC)
   - Calls `runScript("check-doc-freshness", ["--project-root", root], root)`
   - Parses stdout into ValidationResult: count PASS/FAIL lines, extract doc names and versions
   - Returns structured JSON via `{ content: [{ type: "text", text: JSON.stringify(result) }] }`

4. **src/tools/verify-kmp.ts** -- KMP package verification tool:
   - Export `registerVerifyKmpTool(server: McpServer): void`
   - Input schema: `{ projectRoot: string, modulePath?: string, strict?: boolean }`
   - Calls `runScript("verify-kmp-packages", args, toolkitRoot)`
   - Parses output into ValidationResult

5. **src/tools/check-version-sync.ts** -- Version sync tool:
   - Export `registerCheckVersionSyncTool(server: McpServer): void`
   - Input schema: `{ projectRoot?: string }`
   - Calls `runScript("check-version-sync", args, root)`
   - Parses output into ValidationResult

6. **src/tools/script-parity.ts** -- Script parity validation tool:
   - Export `registerScriptParityTool(server: McpServer): void`
   - Input schema: `{ projectRoot?: string }`
   - Instead of wrapping a script, implement directly: read `scripts/sh/` and `scripts/ps1/`, compare basenames (excluding `lib/`), report mismatches
   - Return ValidationResult with details per script

7. **src/tools/setup-check.ts** -- Setup validation tool (locked decision):
   - Export `registerSetupCheckTool(server: McpServer): void`
   - Input schema: `{ projectRoot?: string }`
   - Checks:
     - ANDROID_COMMON_DOC env var is set
     - `docs/` directory exists and has .md files
     - `scripts/sh/` directory exists
     - `scripts/ps1/` directory exists (parity)
     - `skills/` directory exists with SKILL.md files
     - `.claude/agents/` directory exists
   - Return ValidationResult with PASS/FAIL per check
