1. **src/utils/logger.ts** -- stderr-only logger (Pattern 6 from RESEARCH.md):
   - Export `logger` object with info, warn, error (all use console.error), debug (conditional on MCP_DEBUG)
   - NEVER use console.log anywhere

2. **src/utils/paths.ts** -- Toolkit root resolution:
   - Export `getToolkitRoot()`: checks `process.env.ANDROID_COMMON_DOC`, falls back to `path.resolve(new URL(".", import.meta.url).pathname, "../../..")`
   - NOTE: In ES Modules, `__dirname` is not available. Use `import.meta.url` with `fileURLToPath` from `node:url`.
   - On Windows, `fileURLToPath` correctly handles `file:///C:/...` paths
   - Export `getDocsDir()`: `path.join(getToolkitRoot(), "docs")`
   - Export `getSkillsDir()`: `path.join(getToolkitRoot(), "skills")`
   - Export `getScriptsDir()`: `path.join(getToolkitRoot(), "scripts", "sh")`

3. **src/types/results.ts** -- Shared result types:
   - `ValidationResult`: `{ status: "PASS" | "FAIL" | "ERROR" | "TIMEOUT", summary: string, details: ValidationDetail[], duration_ms: number }`
   - `ValidationDetail`: `{ check: string, status: "PASS" | "FAIL" | "WARN", message: string, file?: string }`
   - `ToolResult`: `{ tool: string, result: ValidationResult }`

4. **src/resources/index.ts** -- Stub:

5. **src/tools/index.ts** -- Stub:
   - Export `registerTools(server: McpServer): void` that does nothing (Plan 03 implements)

6. **src/prompts/index.ts** -- Stub:
   - Export `registerPrompts(server: McpServer): void` that does nothing (Plan 02 implements)

7. **src/server.ts** -- Server factory (Pattern 1 from RESEARCH.md):
   - Import and call registerResources, registerTools, registerPrompts
   - Export `createServer()` returning configured McpServer

8. **src/index.ts** -- Entry point:
