  <name>Task 1: Rewrite vault types with ProjectConfig, SubProjectConfig, and layer-first extensions</name>
  <files>mcp-server/src/vault/types.ts</files>
  <action>
Rewrite `mcp-server/src/vault/types.ts` completely. The new type system must support:

**New types to add:**

1. `SubProjectConfig` interface:
   - `name: string` -- display name (e.g., "SessionRecorder-VST3")
   - `path: string` -- relative path from parent project root, or absolute for external sub-projects
   - `collectGlobs?: string[]` -- override collection globs (inherits parent if not set)
   - `excludeGlobs?: string[]` -- override exclusion globs

2. `ProjectConfig` interface:
   - `name: string` -- display name (e.g., "DawSync")
   - `path: string` -- absolute path to project root
   - `layer: "L1" | "L2"` -- layer assignment (L1 for ecosystem libs, L2 for consumer apps)
   - `collectGlobs?: string[]` -- glob patterns for files to collect (relative to project root)
   - `excludeGlobs?: string[]` -- glob patterns to exclude
   - `subProjects?: SubProjectConfig[]` -- optional sub-project definitions
   - `features?: { versionCatalog?: boolean; subProjectScanDepth?: number; }` -- feature flags

**Modified types:**

3. `VaultSourceType` -- add `"architecture"` to the union. Keep existing values: "pattern", "skill", "planning", "claude-md", "agents", "docs", "rule-index", and add "architecture".

4. `VaultSource` -- make `layer` required (not optional). It was `layer?: Layer` -- now `layer: Layer`. Every source must declare its layer. Add optional `subProject?: string` field for sub-project tracking.

5. `VaultEntry` -- add `layer: Layer` field (was missing). Add optional `subProject?: string` field. Keep all existing fields.

6. `VaultConfig` -- breaking rewrite:
   - Add `version: 1` literal type field (for future migrations)
   - Change `projects: string[]` to `projects: ProjectConfig[]`
   - Remove `includeTemplates: boolean` (templates not useful for read-only vault per research)
   - Keep `vaultPath: string`, `autoClean: boolean`, `lastSync?: string`

7. `SyncManifest` -- add `layer` to file tracking: `files: Record<string, { hash: string; sourceType: VaultSourceType; sourcePath: string; layer: Layer; }>`.

8. `SyncResult` -- keep unchanged.

Export all types. Import `Layer` from `"../registry/types.js"` (already done). Add JSDoc comments to all interfaces and fields matching the established documentation style.
