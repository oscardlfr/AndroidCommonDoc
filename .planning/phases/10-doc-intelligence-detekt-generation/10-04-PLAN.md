    - Test: writeGeneratedRules removes orphaned generated files (rule removed from frontmatter)
    - Test: writeGeneratedRules returns GenerationResult with counts of generated, skipped, removed

  <action>
1. Create `mcp-server/src/generation/writer.ts`:
   - Export `GenerationResult = { generated: Array<{ ruleId: string; ruleFile: string; testFile: string }>; skipped: Array<{ ruleId: string; reason: string }>; removed: string[]; providerUpdateBlock: string; configYaml: string; dryRun: boolean }`
   - Export `async function writeGeneratedRules(options: { docsDir: string; rulesOutputDir: string; testsOutputDir: string; dryRun?: boolean }): Promise<GenerationResult>`
     Implementation:
     1. Call `collectAllRules(docsDir)` to get all rule definitions from pattern docs.
     2. For each rule:
        - If `hand_written === true`: add to skipped with reason "hand_written". Continue.
        - Call `emitRule(rule)` to get Kotlin source. If null (unsupported type), skip.
        - Call `emitRuleTest(rule)` to get Kotlin test source. If null, skip.
        - Derive file paths:
          - Rule: `{rulesOutputDir}/{PascalCaseId}Rule.kt`
          - Test: `{testsOutputDir}/{PascalCaseId}RuleTest.kt`
        - If not dryRun: write both files using `fs/promises writeFile`. Create directories if needed with `mkdir({ recursive: true })`.
        - Add to generated array.
     3. Detect orphaned files: list existing .kt files in rulesOutputDir, find any not in the generated set (excluding .gitkeep). If not dryRun, delete them. Add to removed array.
     4. Call `emitRuleSetProviderUpdate(generatedClassNames)` to get the import/registration block.
     5. Call `emitFullConfig(allRules)` to get the config YAML.
     6. Return GenerationResult.

   - Helper: `function toPascalCase(id: string): string` -- converts "prefer-kotlin-time" to "PreferKotlinTime". Split by hyphens, capitalize each word.

2. Write test `writer.test.ts`:
   - Use `os.tmpdir()` + unique subdirectory for isolated file system tests.

     - providerUpdateBlock contains generated class names
     - Orphan detection works (create a stale .kt file, verify it's removed)
   - Test dry-run mode: verify no files are written but result is populated.
   - Clean up temp dirs after tests.

IMPORTANT: Use `path.posix.join` for Kotlin package paths in generated source content. Use `path.join` for filesystem operations. This prevents Windows path separator issues in generated Kotlin code (per 10-RESEARCH.md pitfall 4).
