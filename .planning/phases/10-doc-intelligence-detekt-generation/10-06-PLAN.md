<task type="auto">
  <name>Task 2: Create CI workflow and CLI entrypoint for scheduled monitoring</name>
  <files>.github/workflows/doc-monitor.yml, mcp-server/src/cli/monitor-sources.ts, mcp-server/package.json</files>
  <action>
1. Create `mcp-server/src/cli/monitor-sources.ts`:
   - This is a CLI entrypoint for CI (not an MCP tool -- runs directly via Node.js without MCP transport).
   - Parse command-line arguments:
     - `--tier 1|2|3|all` (default: "all")
     - `--output path/to/report.json` (default: `reports/monitoring-report.json`)
     - `--project-root path` (default: `process.env.ANDROID_COMMON_DOC || path.resolve(__dirname, '../..')`)
   - Implementation:
     1. Scan docs directory for entries with monitor_urls.
     2. Filter by tier if specified.
     3. Call `detectChanges(entries, manifestPath)`.
     4. Load review state. Call `filterNewFindings`.
     5. Call `generateReport`.
     6. Write report JSON to output path (create reports/ directory if needed).
     7. Write summary to stdout for GitHub Actions workflow summary.
     8. Exit with code 0 (monitoring failures are reported as findings, not exit codes -- per 10-RESEARCH.md pitfall 6).
   - Use stderr for logging (logger), stdout for summary output only.

2. Update `mcp-server/package.json`:
   - Add script: `"monitor": "node build/cli/monitor-sources.js"` for convenient CLI invocation.
   - Ensure `build` script compiles the cli/ directory (tsconfig should already include src/**).

3. Create `.github/workflows/doc-monitor.yml`:
   ```yaml
   name: Doc Source Monitoring
   on:
     schedule:
       - cron: '0 9 * * 1'  # Weekly Monday 9am UTC
     workflow_dispatch:
       inputs:
         tier:
           description: 'Monitoring tier (1, 2, 3, or all)'
           default: 'all'
           type: string

   jobs:
     monitor:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: "24"
             cache: "npm"
             cache-dependency-path: mcp-server/package-lock.json
         - run: npm ci
           working-directory: mcp-server
         - run: npm run build
           working-directory: mcp-server
         - name: Run monitoring
           run: node build/cli/monitor-sources.js --tier ${{ inputs.tier || 'all' }} --output reports/monitoring-report.json
           working-directory: mcp-server
           env:
             ANDROID_COMMON_DOC: ${{ github.workspace }}
         - uses: actions/upload-artifact@v4
           if: always()
           with:
             name: monitoring-report-${{ github.run_number }}
             path: mcp-server/reports/
             retention-days: 30
   ```
   - CI outputs artifact (not GitHub Issues) per discretion recommendation in 10-RESEARCH.md.
   - `workflow_dispatch` allows manual triggering with tier selection.
   - Exit code 0 always (findings are data, not errors) -- workflow failure only on infrastructure issues.

IMPORTANT: CLI entrypoint uses stderr-only for logging (no console.log to stdout) -- same pattern as the MCP server to prevent output corruption.
  </action>
