1. Create `mcp-server/src/tools/generate-detekt-rules.ts`:
   - Export `registerGenerateDetektRulesTool(server: McpServer, limiter?: RateLimiter): void`
   - Register tool "generate-detekt-rules" with:
     - title: "Generate Detekt Rules"
     - description: "Generate Detekt custom rules from pattern doc frontmatter rule definitions. Reads rules: YAML from pattern docs and emits Kotlin source files for AST-only Detekt rules with companion tests."
     - inputSchema using Zod:
       - `dry_run`: z.boolean().optional().default(true) -- "Preview changes without writing files (default: true for safety)"
       - `projectRoot`: z.string().optional() -- "Path to toolkit root"
       - `consumer_project`: z.string().optional() -- "Path to consumer project for L1 rule generation"
     - Implementation:
       1. Check rate limit.
       2. Resolve toolkit root. Determine docsDir, rulesOutputDir, testsOutputDir:
          - For toolkit: `{root}/docs/`, `{root}/detekt-rules/src/main/kotlin/com/androidcommondoc/detekt/rules/generated/`, `{root}/detekt-rules/src/test/kotlin/com/androidcommondoc/detekt/rules/generated/`
          - For consumer (if consumer_project provided): scan consumer's `.androidcommondoc/docs/` for L1 rules
       3. Call `writeGeneratedRules({ docsDir, rulesOutputDir, testsOutputDir, dryRun })`.
       4. Return JSON with GenerationResult + instructions for next steps (compile with `./gradlew :detekt-rules:test`, update RuleSetProvider if new rules added).

2. Create `mcp-server/src/tools/ingest-content.ts`:
   - Export `registerIngestContentTool(server: McpServer, limiter?: RateLimiter): void`
   - Register tool "ingest-content" with:
     - title: "Ingest Content"
     - description: "Analyze content from any source (URL, pasted text) and extract relevant patterns for routing to pattern docs. Supports Medium posts, LinkedIn articles, blog posts, conference talks. When a URL is unfetchable (paywall, auth-wall), prompts for pasted content."
     - inputSchema using Zod:
       - `url`: z.string().optional() -- "URL to fetch content from (optional if content is provided)"
       - `content`: z.string().optional() -- "Pasted text content (for unfetchable URLs or direct input)"
       - `projectRoot`: z.string().optional() -- "Path to toolkit root"
     - Implementation:
       1. Check rate limit.
       2. Validate input: either url or content must be provided.
       3. If url provided:
          - Try to fetch the URL with a 15-second timeout.
          - If fetch fails (403, 401, timeout, network error): return a structured response with `status: "url_unfetchable"` and `suggestion: "The URL could not be fetched (possible paywall or auth-wall). Please paste the content directly using the 'content' parameter."`. Log the unsupported URL type for future support tracking.
          - If fetch succeeds: use the response text as the content.
       4. Extract patterns from content:
          - Scan for keyword matches against existing pattern doc metadata (scope, sources fields from registry entries).
          - Identify version references, library mentions, API recommendations.
          - Group findings by which pattern doc they relate to.
       5. Return structured JSON:
          - `status: "analyzed"` or `"url_unfetchable"`
          - `suggestions`: Array of `{ target_doc: string; slug: string; relevance: string; extracted_patterns: string[]; recommended_action: "update" | "review" | "new_doc" }`
          - `raw_content_preview`: First 500 chars of analyzed content.
          - NEVER auto-apply changes. All suggestions require user review.

3. Write tests:
   - `generate-detekt-rules.test.ts`: Mock writeGeneratedRules via vi.mock. Test tool registration, basic invocation, dry_run behavior.
   - `ingest-content.test.ts`: Mock fetch and scanDirectory. Test URL fetch success, URL fetch failure (graceful degradation with paste prompt), content analysis with pattern matching, empty content handling.
