  <name>Task 1: Build review state tracking with persistence and TTL</name>
  <files>mcp-server/src/monitoring/review-state.ts, mcp-server/src/monitoring/report-generator.ts, mcp-server/tests/unit/monitoring/review-state.test.ts, mcp-server/tests/unit/monitoring/report-generator.test.ts</files>
  <behavior>
    - Test: loadReviewState returns empty state when file doesn't exist
    - Test: saveReviewState writes valid JSON and loadReviewState reads it back
    - Test: saveReviewState uses atomic write (write to temp, rename)
    - Test: filterNewFindings removes findings whose hash matches an accepted/rejected entry
    - Test: filterNewFindings re-surfaces deferred findings past TTL (default 90 days)
    - Test: filterNewFindings keeps deferred findings within TTL filtered out
    - Test: generateReport groups findings by severity and includes tier information
    - Test: generateReport includes summary counts (total, new, high, medium, low)
  </behavior>
  <action>
1. Create `mcp-server/src/monitoring/review-state.ts`:
   - Export `ReviewStatus = "accepted" | "rejected" | "deferred"`
   - Export `ReviewEntry = { status: ReviewStatus; reviewed_at: string; finding_hash: string; reason?: string; action?: string; deferred_until?: string }`
   - Export `ReviewState = { schema_version: 1; last_run: string; findings: Record<string, ReviewEntry> }`
   - Export `async function loadReviewState(statePath: string): Promise<ReviewState>`
     - Read JSON file at statePath. If doesn't exist, return default empty state.
     - Validate schema_version === 1. If different, log warning and return empty state.
   - Export `async function saveReviewState(statePath: string, state: ReviewState): Promise<void>`
     - Atomic write: write to `${statePath}.tmp`, then rename to statePath using `fs/promises` rename.
     - Pretty-print JSON with 2-space indent.
   - Export `function filterNewFindings(findings: MonitoringFinding[], state: ReviewState, ttlDays?: number): MonitoringFinding[]`
     - Default ttlDays = 90
     - For each finding, check if finding_hash exists in state.findings:
       - If status === "accepted" or "rejected": filter out (already reviewed)
       - If status === "deferred": check if deferred_until or reviewed_at + ttlDays has passed. If expired, re-surface (keep in output as "stale deferral"). If not expired, filter out.
     - Return only new/re-surfaced findings.

2. Create `mcp-server/src/monitoring/report-generator.ts`:
   - Export `MonitoringReport = { timestamp: string; tier_filter: MonitoringTier | "all"; total_sources: number; checked: number; errors: number; findings: { total: number; new: number; high: number; medium: number; low: number }; details: MonitoringFinding[]; stale_deferrals: string[] }`
   - Export `function generateReport(changeReport: ChangeReport, newFindings: MonitoringFinding[], tierFilter: MonitoringTier | "all", staleDeferrals: string[]): MonitoringReport`
     - Aggregate counts by severity from newFindings.
     - Include stale_deferrals list (finding hashes that were deferred but expired).

3. Write tests:
   - `review-state.test.ts`: Use `os.tmpdir()` for temp file paths. Test load/save round-trip, atomic write (verify .tmp file is cleaned up), filterNewFindings with accepted/rejected/deferred entries, TTL expiry logic.
   - `report-generator.test.ts`: Create mock ChangeReport and newFindings, verify report structure and severity counts.
  </action>

  </verify>
