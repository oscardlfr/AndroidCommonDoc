  <name>Task 2: Build source-checker and change-detector monitoring engine</name>
  <files>mcp-server/src/monitoring/source-checker.ts, mcp-server/src/monitoring/change-detector.ts, mcp-server/tests/unit/monitoring/source-checker.test.ts, mcp-server/tests/unit/monitoring/change-detector.test.ts</files>
  <behavior>
    - Test: checkSource with type "github-releases" extracts latest version from mock GitHub API response
    - Test: checkSource with type "doc-page" returns SHA-256 content hash
    - Test: checkSource with type "maven-central" extracts latest version from mock Maven API response
    - Test: checkSource handles HTTP errors (404, 429, timeout) gracefully with categorized failure
    - Test: detectChanges identifies version drift when upstream version > manifest version
    - Test: detectChanges flags deprecation mentions as HIGH severity
    - Test: detectChanges returns empty findings when no changes detected
    - Test: finding_hash is deterministic for the same finding data
  </behavior>
  <action>
1. Create `mcp-server/src/monitoring/source-checker.ts`:
   - Export `CheckResult = { url: string; type: MonitorUrlType; status: "ok" | "error" | "unreachable"; content_hash?: string; latest_version?: string; raw_content?: string; error?: string; fetched_at: string }`
   - Export `async function checkSource(monitorUrl: MonitorUrl): Promise<CheckResult>`
   - For `github-releases`: Use fetch() to call `https://api.github.com/repos/{owner}/{repo}/releases/latest` (extract owner/repo from the URL pattern). Parse JSON, extract `tag_name` as latest_version. Strip leading "v" from version tag.
   - For `maven-central`: Use fetch() to call `https://search.maven.org/solrsearch/select?q=a:{artifactId}&rows=1&wt=json`. Parse response, extract latestVersion.
   - For `doc-page`: Use fetch() to GET the URL. Compute SHA-256 hash of response body using `node:crypto` createHash. Return as content_hash.
   - For all types: Handle HTTP errors gracefully. On 404: status "error". On 429: status "unreachable" (transient). On timeout (use AbortController with 15s): status "unreachable". On network error: status "unreachable".
   - Log each check via logger.info (URL being checked) and logger.warn (on errors).

2. Create `mcp-server/src/monitoring/change-detector.ts`:
   - Export `ChangeReport = { findings: MonitoringFinding[]; checked: number; errors: number; timestamp: string }`
   - Export `async function detectChanges(entries: RegistryEntry[], manifestPath?: string): Promise<ChangeReport>`
   - For each entry with `monitor_urls`, call `checkSource` for each URL.
   - Compare results against `versions-manifest.json`:
     - If checkResult.latest_version exists and differs from manifest version for that source: create MEDIUM finding ("version drift")
     - If checkResult.raw_content contains deprecation keywords ("deprecated", "removed", "breaking change"): create HIGH finding ("deprecation detected")
     - If checkResult.content_hash differs from previously stored hash: create MEDIUM finding ("content changed")
   - Generate deterministic `finding_hash` using SHA-256 of `slug + url + category + summary`.
   - Handle the case where versions-manifest.json doesn't exist or doesn't contain the relevant library gracefully (skip version comparison, log warning).

3. Create test files with mock fetch (use Vitest `vi.fn()` and `vi.stubGlobal('fetch', ...)` to mock global fetch):
   - `source-checker.test.ts`: Mock fetch responses for each URL type. Test success and error paths.
   - `change-detector.test.ts`: Create mock RegistryEntry objects with monitor_urls. Mock checkSource results. Verify findings generation.

IMPORTANT: Use `node:crypto` for hashing (built-in, no dependencies). Use global `fetch` (available in Node 18+). All monitoring modules go in `mcp-server/src/monitoring/` directory.
  </action>
