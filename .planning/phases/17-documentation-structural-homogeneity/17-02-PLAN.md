  <name>Task 1: Implement validate-vault tool with 4 validation dimensions</name>
  <files>mcp-server/src/tools/validate-vault.ts, mcp-server/src/tools/index.ts</files>
  <action>
Create `validate-vault.ts` following the existing validation tool pattern (pure exported functions + MCP registration).

Implement 4 validation dimensions as separate exported functions:

1. `checkDuplicates(vaultPath: string): Promise<ValidationIssue[]>` -- Scan all .md files in vault. Detect: (a) case-insensitive filename collisions within same directory, (b) files with identical content hash (SHA-256) across directories, (c) same vault_source frontmatter pointing to multiple files. Return list of duplicate pairs with paths.

2. `checkStructuralHomogeneity(projectPaths: ProjectPath[]): Promise<ValidationIssue[]>` -- For each project's docs/ directory: (a) every subdirectory with 2+ .md files must have a hub doc (contains "## Sub-documents" heading), (b) hub docs must be <100 lines, (c) sub-docs must be <300 lines, (d) all files must have 10-field frontmatter (scope, sources, targets, slug, status, layer, category, description, version, last_updated). Takes array of {name, path, layer} for L0/L1/L2 projects.

3. `checkReferenceIntegrity(projectPaths: ProjectPath[]): Promise<ValidationIssue[]>` -- Scan L1 docs for l0_refs fields; verify each referenced L0 slug exists in L0 docs/. Scan L2 docs for l0_refs and l1_refs; verify they exist. Check that no L0 doc references L1/L2 by name (upward reference violation). Report broken refs with source file, ref slug, and expected location.

4. `checkWikilinkCoverage(vaultPath: string): Promise<ValidationIssue[]>` -- Scan vault .md files for `[[wikilink]]` syntax. For each wikilink, verify the target file exists in the vault. Report: broken wikilinks (target not found), orphan files (no incoming wikilinks and not a MOC page).

Orchestrator function `validateVault(params)` runs all 4 checks, aggregates results:
```typescript
interface ValidateVaultResult {
  duplicates: ValidationIssue[];
  homogeneity: ValidationIssue[];
  references: ValidationIssue[];
  wikilinks: ValidationIssue[];
  summary: { errors: number; warnings: number; passed: boolean };
}
```

Register as MCP tool "validate-vault" with Zod schema accepting:
- `vaultPath` (optional, defaults to configured vault path)
- `checks` (optional array of check names, defaults to all 4)
- `projectPaths` (optional, defaults to discovered projects)

Register in `index.ts`: import and call `registerValidateVaultTool(server, rateLimiter)`. Update logger message to "Registered 17 tools".
