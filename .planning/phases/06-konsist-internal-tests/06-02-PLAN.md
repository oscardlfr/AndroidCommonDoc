    Tests:
    1. **`detekt-rules classes reside in com_androidcommondoc_detekt package`**
       - `ScopeFactory.detektRulesScope().classes()` should all have package starting with `com.androidcommondoc.detekt`
       - Use `.assertTrue(additionalMessage = "All detekt-rules classes must be in com.androidcommondoc.detekt or a subpackage. Move the class to the correct package.") { it.resideInPackage("com.androidcommondoc.detekt..") }`

    2. **`build-logic classes reside in com_androidcommondoc_gradle package`**
       - `ScopeFactory.buildLogicScope().classes()` should all have package starting with `com.androidcommondoc.gradle`
       - Same pattern with `.resideInPackage("com.androidcommondoc.gradle..")`

    3. **`no detekt-rules class uses build-logic packages`** (module isolation)
       - `ScopeFactory.detektRulesScope().files` should NOT have imports starting with `com.androidcommondoc.gradle`

    4. **`no build-logic class uses detekt rules packages`** (reverse module isolation)
       - `ScopeFactory.buildLogicScope().files` should NOT have imports starting with `com.androidcommondoc.detekt.rules`
       - additionalMessage: "build-logic must not directly import detekt rule implementations. Use the JAR artifact via Gradle dependency instead."

    Validates class suffix conventions match package expectations (KONS-03).

    Tests:

       - Assert all names end with "Rule"

    2. **`classes ending with Rule reside in a rules package`** (reverse check)
       - Filter: `detektRulesScope().classes().withNameEndingWith("Rule")`
