# Plan 19-01: Upstream Validation + Detekt Rules

**Phase:** 19 Android CLI Integration
**Wave:** 1 (parallel with 19-POC)
**Depends on:** none
**Blast radius:** LOW
**Requirements:** AC-CLI-01

## Goal

`android docs search/fetch kb://...` becomes an additional source for upstream validation alongside Context7/WebFetch. New Compose doc generates a Detekt rule.

## Scope

### Extend upstream validation
- `docs/guides/upstream-validation.md` — add section "Android CLI as upstream source" describing `kb://` URIs, online-only mode, and fallback when `android` is not on PATH
- `skills/validate-upstream/SKILL.md` — add Wizard step for source `android-cli`
- `mcp-server/src/tools/audit-docs.ts` — add `source_mode: "android-cli" | "webfetch"` param
- `mcp-server/src/monitoring/content-fetcher.ts` — adapter shells out to `android docs fetch <uri>`

### New Compose doc + generated Detekt rule
- `docs/compose/compose-layout-validation.md` with `rules:` frontmatter describing layout anti-patterns
- `generate-detekt-rules` MCP tool emits `ComposeLayoutValidationRule.kt` in `detekt-rules/src/main/kotlin/.../rules/`

## Success Criteria

1. `/validate-upstream --source=android-cli <doc>` returns PASS/FAIL equivalent to `--source=webfetch` on the same doc (±1 tolerance for cache staleness)
2. `audit-docs` MCP tool with `source_mode: "android-cli"` and network disabled emits clear error referencing offline mode (not cryptic exit code)
3. `generate-detekt-rules` emits `ComposeLayoutValidationRule.kt` with unit test passing from frontmatter examples
4. `docs/compose/compose-layout-validation.md` validates via `validate-doc-structure` MCP tool (all required frontmatter fields present)
5. `cd mcp-server && npm test` green
6. `./gradlew :detekt-rules:test` green
