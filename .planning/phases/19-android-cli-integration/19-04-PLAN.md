## Scope

### Narrow MCP bridge (stateful commands only)
- New `mcp-server/src/tools/android-cli-bridge.ts` scoped ONLY to two destructive commands:
  - `android run --apks=...` (deploys APK) — validates args, logs target device/serial
  - `android create` (writes files) — validates template name, enforces output path within project
- Register in `mcp-server/src/tools/index.ts`; update counter "40 tools" → "41 tools"
- Bash permissions for read-only commands: `.claude/settings.json` template in `setup/` with allowlist:
  - `Bash(android:docs:*)`
  - `Bash(android:screen:*)`
  - `Bash(android:layout:*)`
  - `Bash(android:skills:list)`
  - `Bash(android:sdk:list)`
- New `docs/guides/agent-tool-permissions.md` listing which `android` commands go through Bash vs. bridge

### CI Linux workflow
- New `.github/workflows/ui-validation.yml` executing `android-layout-diff` in Linux with emulator for PRs touching UI modules
- Trigger on `pull_request` for paths `**/ui/**`, `**/*Screen.kt`, `**/*Compose*.kt`
- `l0-manifest.json` schema — new optional field `android_cli.ui_validation_policy: "physical" | "ci-only" | "disabled"` for consumers to opt out

## Risks & Mitigations

- **CI emulator cost/time**: Linux emulator adds ~5 min per PR. Mitigation: scope workflow to UI-only paths via path filters.
- **Consumer manifest migration**: `android_cli.ui_validation_policy` is optional; default inferred from host OS.
- **Rollback plan**: if `/sync-l0` to consumers introduces unexpected churn, revert by deleting Phase 19 skills + tools from consumer `.claude/` dirs and re-running sync with `--prune`.
