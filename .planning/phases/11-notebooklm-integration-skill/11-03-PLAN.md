Create `mcp-server/src/vault/moc-generator.ts`:

- Each MOC generator function takes `VaultEntry[]` and returns a `VaultEntry` (a MOC page is itself a VaultEntry).

- `generateAllPatternsMOC(entries: VaultEntry[]): VaultEntry` -- filters entries by sourceType "pattern", groups by scope tags. Frontmatter: `{ tags: ["moc", "index"], vault_type: "moc", vault_synced: today }`. Content: `# All Patterns\n\n## By Scope\n### {scope}\n- [[{slug}]]\n...`. vaultPath: "00-MOC/All Patterns.md".

- `generateByProjectMOC(entries: VaultEntry[]): VaultEntry` -- groups all entries by project (default "AndroidCommonDoc" when project is undefined). Content: `# By Project\n\n## {project}\n- [[{slug}]] ({sourceType})\n...`. vaultPath: "00-MOC/By Project.md".

- `generateByLayerMOC(entries: VaultEntry[]): VaultEntry` -- groups pattern entries by layer tag (L0, L1, L2). vaultPath: "00-MOC/By Layer.md".

- `generateByPlatformMOC(entries: VaultEntry[]): VaultEntry` -- groups entries by target platform tags (android, desktop, ios, macos, web, jvm). vaultPath: "00-MOC/By Target Platform.md".

- `generateAllSkillsMOC(entries: VaultEntry[]): VaultEntry` -- filters sourceType "skill". vaultPath: "00-MOC/All Skills.md".

- `generateAllDecisionsMOC(entries: VaultEntry[]): VaultEntry` -- filters sourceType "planning" (decisions). vaultPath: "00-MOC/All Decisions.md".

- `generateAllMOCs(entries: VaultEntry[]): VaultEntry[]` -- calls all 6 generators, returns the array.

MOC pages use static [[wikilinks]], no Dataview queries. Each MOC has a brief description paragraph after the heading.

Create `mcp-server/src/vault/vault-writer.ts`:

- `generateObsidianConfig(): Record<string, string>` -- returns JSON strings for app.json (useMarkdownLinks: false, showFrontmatter: true), community-plugins.json (["dataview"]), core-plugins.json (file-explorer, search, graph, backlink, outgoing-link, tag-pane, page-preview, templates, command-palette enabled), graph.json (color groups: MOC=gold, patterns=green, skills=blue, projects=orange).

- `loadManifest(vaultPath: string): Promise<SyncManifest>` -- reads `_vault-meta/sync-manifest.json`, returns empty manifest `{ lastSync: "", files: {} }` if not found.

- `saveManifest(vaultPath: string, manifest: SyncManifest): Promise<void>` -- writes manifest to `_vault-meta/sync-manifest.json`.

- `detectOrphans(manifest: SyncManifest, currentPaths: Set<string>): string[]` -- returns array of manifest file paths not in currentPaths.

- `computeHash(content: string): string` -- uses `createHash("sha256").update(content).digest("hex").slice(0, 16)` from `node:crypto` for compact hashes.

- `writeVault(vaultPath: string, entries: VaultEntry[], options: { init?: boolean; clean?: boolean }): Promise<SyncResult>` -- orchestrates file writing:
  1. Load existing manifest
  2. If `init`: create .obsidian/ directory with config files, create _vault-meta/README.md
  3. For each entry: compute hash, compare with manifest. If different or new: mkdir -p for parent dir, write file. If same: increment unchanged.
  4. Build new manifest from all written + unchanged files
  5. If `clean`: detect orphans, delete them, increment removed
  6. Save updated manifest
  7. Return SyncResult with counts and duration

Use `node:crypto` for hashing (built-in, no dependency). Use `mkdir(dir, { recursive: true })` for directory creation. Normalize all paths with forward slashes.

The _vault-meta/README.md content should explain: this vault is auto-generated by AndroidCommonDoc's sync-vault tool, repos are source of truth, do not edit files directly (changes will be overwritten on next sync), and link to the skill documentation.

Create `mcp-server/src/vault/sync-engine.ts` -- the orchestrator that ties the pipeline together:

- `syncVault(configOverride?: Partial<VaultConfig>): Promise<SyncResult>`
  1. Load config via `loadVaultConfig()`, merge with configOverride if provided
  2. Collect all sources via `collectAll(config)`
  3. Transform sources via `transformAll(sources)`
  4. Generate MOC pages via `generateAllMOCs(entries)`
  5. Combine entries + MOCs
  6. Write to vault via `writeVault(config.vaultPath, allEntries, { init: false })`
  7. Update config.lastSync and save via `saveVaultConfig()`
  8. Return SyncResult

- `initVault(configOverride?: Partial<VaultConfig>): Promise<SyncResult>`
  1. Load config, merge with configOverride
  2. Same pipeline as syncVault but calls `writeVault` with `{ init: true }`
  3. Creates the vault directory if it doesn't exist
  4. Writes .obsidian/ config files
  5. Returns SyncResult

- `cleanOrphans(configOverride?: Partial<VaultConfig>): Promise<SyncResult>`
  1. Load config, merge
  2. Same pipeline but calls `writeVault` with `{ init: false, clean: true }`
  3. Returns SyncResult (removed count reflects cleaned orphans)

- `getVaultStatus(configOverride?: Partial<VaultConfig>): Promise<{ configured: boolean; vaultPath: string; lastSync: string | null; fileCount: number; orphanCount: number; projects: string[] }>`
  1. Load config
  2. Check if vault directory exists
  3. Load manifest, count files
  4. Collect current sources to detect potential orphans
  5. Return status object (cheap, no actual writing)

All functions handle errors gracefully: if vault path doesn't exist for status, return configured: false. If collect fails for a project, log warning and continue with other projects.

Import logger from `../utils/logger.js` for error reporting (stderr only, consistent with MCP server pattern).
