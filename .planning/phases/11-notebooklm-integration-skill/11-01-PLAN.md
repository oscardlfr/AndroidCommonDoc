Create `mcp-server/src/vault/collector.ts`:

- `collectPatternDocs(toolkitRoot: string): Promise<VaultSource[]>` -- uses `scanDirectory(docsDir, "L0")` to find pattern docs, reads each file's content, creates VaultSource with sourceType "pattern", layer "L0", relativePath "patterns/{slug}.md". Extracts metadata from RegistryEntry.

- `collectL1Docs(projectPath: string, projectName: string): Promise<VaultSource[]>` -- uses `scanDirectory(l1DocsDir, "L1", projectName)` for project-specific pattern overrides at `.androidcommondoc/docs/`. Sets relativePath to "patterns-L1/{projectName}/{slug}.md".

- `collectSkills(toolkitRoot: string): Promise<VaultSource[]>` -- reads `skills/` directory, for each subdirectory checks for SKILL.md, reads content, creates VaultSource with sourceType "skill", relativePath "skills/{dirname}.md". Metadata is null (skills have their own format).

- `collectProjectKnowledge(projectPath: string, projectName: string): Promise<VaultSource[]>` -- collects from a single project:
  - CLAUDE.md (if exists) -> sourceType "claude-md", relativePath "projects/{name}/CLAUDE.md"
  - .planning/PROJECT.md (if exists) -> sourceType "planning", relativePath "projects/{name}/PROJECT.md"
  - .planning/STATE.md decisions section (if exists) -> sourceType "planning", relativePath "projects/{name}/decisions.md"
  - AGENTS.md: if >500 lines, set a `largefile: true` property; still collect the content but flag it
  - Skip files that don't exist (try/catch readFile, continue on ENOENT)

- `collectAll(config: VaultConfig): Promise<VaultSource[]>` -- orchestrates full collection:
  1. Get toolkitRoot via `getToolkitRoot()`
  2. Collect patterns + skills from toolkit root
  3. Collect project knowledge for AndroidCommonDoc itself (toolkit root)
  4. If config.projects is empty, call `discoverProjects()` to auto-discover. If config.projects is non-empty, build ProjectInfo[] from names (resolve paths from parent of toolkit root).
  5. For each project: collectProjectKnowledge + collectL1Docs (if hasL1Docs)
  6. Also collect shared-kmp-libs docs if it's in the discovered/configured projects
  7. Return combined array

For tests, use temp directories with fixture files (mkdtemp + writeFile in beforeEach, rm in afterEach). Mock `discoverProjects` and `scanDirectory` for unit isolation by using vi.mock. Mock `getToolkitRoot` to point to the temp directory.

Normalize all paths with forward slashes using `.replace(/\\/g, '/')` for Windows compatibility in relativePath fields.
