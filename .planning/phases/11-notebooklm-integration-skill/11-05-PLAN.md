Frontmatter:
```yaml
---
name: sync-vault
description: "Sync KMP ecosystem documentation into a unified Obsidian knowledge vault with auto-generated wikilinks, MOC pages, and tags"
metadata:
  author: AndroidCommonDoc
  version: "1.0"
  params:
    - project-root
---
```

Sections:

**Usage Examples:**
```
/sync-vault --init          # First time: create vault + .obsidian config + initial sync
/sync-vault                 # Update existing vault with latest changes
/sync-vault --status        # Check vault health without modifying
/sync-vault --clean         # Remove orphaned files no longer in source repos
```

**Parameters:** Uses project-root from params.json. Additional skill-specific arguments:
- `--init` -- Initialize a new vault (creates directory, .obsidian config, syncs all sources)
- `--status` -- Show vault health info (last sync, file count, orphan warnings)
- `--clean` -- Remove vault files that no longer exist in source repos
- `--vault-path <path>` -- Override default vault location (~/.../kmp-knowledge-vault)

**Behavior:** 6 steps:
1. If --init: call sync-vault MCP tool with mode "init" to create vault directory, write .obsidian config, perform initial full sync.
2. If --status: call vault-status MCP tool and display health info.
3. If no flags (default): call sync-vault MCP tool with mode "sync" to update vault.
4. If --clean: call sync-vault MCP tool with mode "clean" to remove orphaned files.
5. Display sync results (written, unchanged, removed counts).
6. If first run, remind user to open the vault in Obsidian and install recommended community plugins (Dataview).

**Implementation:** orchestration workflow using MCP tools. Steps: call sync-vault/vault-status tool, parse JSON response, present results to user.

**Expected Output:** Show sync result summary with counts.

**Cross-References:** Link to sync-vault MCP tool, vault-status MCP tool, existing find-pattern tool, monitor-docs skill, vault config at ~/.androidcommondoc/vault-config.json, vault location, _vault-meta/SETUP.md.
