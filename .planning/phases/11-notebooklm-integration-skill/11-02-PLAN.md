Create `mcp-server/src/vault/tag-generator.ts`:
- `generateTags(params: { scope?: string[]; targets?: string[]; layer?: string; sourceType: VaultSourceType; project?: string }): string[]`
- Collects tags from: scope values, target values, layer (if present), sourceType, project name (if present)
- Deduplicates via Set, sorts alphabetically, returns string[]
- All tags lowercase for consistency

Create `mcp-server/src/vault/wikilink-generator.ts`:
- `injectWikilinks(content: string, allSlugs: Set<string>, ownSlug?: string): string`
- For each slug in allSlugs (skip ownSlug to prevent self-links):
  - Split content into "safe" and "unsafe" zones. Unsafe zones: inside `[[...]]`, inside single backticks, inside fenced code blocks (``` ... ```). Only replace in safe zones.
  - Strategy: use a zone-based approach. First, identify all "protected" ranges (wikilinks, code spans, code blocks). Then, for remaining text, replace standalone slug occurrences with `[[slug]]`.
  - Match slug as a whole word (word boundaries) to avoid partial matches.
  - Use `.replace(/\\/g, '/')` on any paths before they become wikilinks (Windows pitfall).
- Return the modified content string.

For the zone-based protection, split content by fenced code blocks first (split on triple backtick lines), then within non-code sections protect inline code and existing wikilinks. This avoids complex regex with lookaheads.

  <action>
Create `mcp-server/src/vault/transformer.ts`:

- `transformSource(source: VaultSource, allSlugs: Set<string>): VaultEntry`
  1. Parse existing frontmatter from source.content using `parseFrontmatter()` from `../registry/frontmatter.js`
  2. Extract body content (everything after frontmatter)
  3. Determine slug from relativePath: `path.basename(source.relativePath, ".md")`
  4. Generate tags via `generateTags()` using metadata fields + source.layer + source.sourceType + source.project
  5. Build enriched frontmatter object:
     - Spread all original frontmatter fields
     - Add: `tags` (generated), `aliases` (array with slug), `vault_source` (source.filepath relative, normalized forward slashes), `vault_synced` (today's date YYYY-MM-DD), `vault_type` (map sourceType: pattern->"pattern", skill->"skill", planning->"planning", claude-md->"reference", agents->"reference", docs->"reference")
  6. Inject wikilinks in body via `injectWikilinks(body, allSlugs, slug)`
  7. Return VaultEntry with: slug, vaultPath (source.relativePath), content (rebuilt with enriched frontmatter YAML + body), frontmatter, sourceType, project, tags

- `transformAll(sources: VaultSource[]): VaultEntry[]`
  1. Collect all slugs: `new Set(sources.map(s => path.basename(s.relativePath, ".md")))`
  2. Map each source through `transformSource(source, allSlugs)`
  3. Return VaultEntry[]

Use `yaml.stringify()` from the `yaml` package to serialize enriched frontmatter (already a dependency). Wrap with `---\n` delimiters.

Handle null metadata gracefully: if source.metadata is null, create minimal enriched frontmatter with just vault_* fields and tags from sourceType/project/layer.
