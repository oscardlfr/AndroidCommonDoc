For EACH file, produce an audit entry with these fields:
- `path`: Relative path from WakeTheCave root (e.g., `docs/01-architecture/patterns/viewmodel-patterns.md`)
- `directory`: Parent subdirectory for grouping (e.g., `docs/01-architecture/patterns`)
- `current_layer`: Always "L2" (WakeTheCave is an app project)
- `recommended_layer`: "L0" if pattern is clearly sourced from official docs (Kotlin, Jetpack, KMP, Gradle); "L1" if it documents shared-kmp-libs conventions; "L2" if domain-specific or architectural insight
- `ai_readiness_score`: 0-5 using standard criteria: +1 has YAML frontmatter, +1 section size <=150 lines (split by ## headings, count per section), +1 proper subdivision by concern (multiple ## sections, each focused), +1 no stale version refs (extract version patterns, compare against versions-manifest.json), +1 actionable content (code examples, specific instructions, structured tables -- not just prose)
- `ai_readiness_breakdown`: Object with boolean for each of the 5 criteria
- `classification`: "ACTIVE" if content is current and useful; "SUPERSEDED" if the file explicitly points elsewhere (like viewmodel-patterns.md pointing to AndroidCommonDoc) or content is replaced by another doc; "UNIQUE" if contains irreplaceable domain knowledge
- `l0_candidate`: boolean -- true ONLY if recommended_layer is "L0" AND the content is clearly sourced from official documentation (conservative threshold per user decision)
- `l0_rationale`: If l0_candidate is true, explain what official source the pattern derives from and why it qualifies
- `action`: "promote_L0" | "flag_L1_review" | "keep_L2" | "advisory_superseded"
- `overlaps_with`: Array of known AndroidCommonDoc pattern doc slugs this file overlaps with (e.g., ["error-handling-patterns", "viewmodel-patterns"])
- `notes`: Any advisory observations (e.g., "deprecated, points to AndroidCommonDoc", "valuable domain insight", "archive candidate")

CRITICAL classification rules:
- L0 promotion: ONLY if pattern clearly comes from official Kotlin/Jetpack/KMP/Gradle documentation. If uncertain, keep as L2.
- WakeTheCave-derived architectural insights stay L2 candidates for manual review (per user decision).
- docs/ archive subdirectory: Check each file -- most are SUPERSEDED but some may contain UNIQUE content.
- docs2/ files: These have a clean Architecture-layer structure -- assess whether they document generic patterns (L0) or WakeTheCave-specific architecture (L2).
- Check for files that already point to AndroidCommonDoc as canonical (e.g., viewmodel-patterns.md).

Version freshness checking: For each file, extract version references using patterns like `kotlin\s+(\d+\.\d+\.\d+)`, `compose[-\s]multiplatform\s+(\d+\.\d+\.\d+)`, `koin\s+(\d+\.\d+\.\d+)`, etc. Compare against versions-manifest.json entries.
