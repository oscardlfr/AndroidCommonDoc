For EACH file, produce an audit entry:
- `path`: Relative path from DawSync root
- `directory`: Category directory for grouping
- `current_layer`: Always "L2" (DawSync is an app project)
- `recommended_layer`:
  - "L0" if the pattern is generic KMP/Android from official docs (conservative, same threshold as WakeTheCave)
  - "L1" if it documents shared-kmp-libs conventions or ecosystem patterns
  - "L2" for domain-specific content (DAW, VST3, session management, audio, DawSync-specific architecture)
- `ai_readiness_score`: 0-5 using standard criteria (same as WakeTheCave plan)
- `ai_readiness_breakdown`: Object with boolean for each of the 5 criteria
- `classification`:
  - "ACTIVE": Content is current, relevant, and in use
  - "SUPERSEDED": Content exists elsewhere or is replaced (e.g., a doc in docs/archive/ that has been superseded by an updated version, or a pattern now in AndroidCommonDoc)
  - "UNIQUE": Contains irreplaceable domain context (DawSync-specific knowledge not available elsewhere)
- `action`: "keep" | "promote_L0" | "promote_L1" | "consolidate" | "archive" | "flag_l2_override"
- `l2_override`: boolean -- true if this DawSync pattern SHOULD override the equivalent shared-kmp-libs (L1) pattern. Per user decision: "Some DawSync patterns should override shared-kmp-libs"
- `l2_override_target`: If l2_override is true, which L1 pattern it overrides (e.g., "testing-strategy")
- `overlaps_with`: Array of known AndroidCommonDoc pattern doc slugs or shared-kmp-libs doc names this file overlaps with
- `versionRefs`: Array of { tech, found, current, stale } for version references found in the file
- `notes`: Observations, consolidation recommendations, etc.

- docs/archive/ (~79 files, most dated 2026-01): Default to SUPERSEDED but check each for UNIQUE content. Some archived docs may contain irreplaceable domain context.
