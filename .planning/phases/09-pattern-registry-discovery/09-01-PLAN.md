  <files>

  <behavior>
    Frontmatter parser:
    - parseFrontmatter("---\nscope:\n  - testing\n---\n# Content") returns { data: { scope: ["testing"] }, content: "# Content" }
    - parseFrontmatter("# No frontmatter") returns null
    - parseFrontmatter("---\n---\n# Empty FM") returns { data: {}, content: "# Content" }
    - parseFrontmatter with BOM (\uFEFF prefix) strips BOM and parses correctly
    - parseFrontmatter with CRLF line endings parses correctly
    - parseFrontmatter with invalid YAML returns null (not throw)

    Scanner:
    - scanDirectory(docsDir) returns RegistryEntry[] with one entry per .md file that has valid frontmatter
    - scanDirectory skips files without frontmatter (returns empty metadata)
    - scanDirectory skips non-.md files
    - Each RegistryEntry has slug derived from filename (minus .md), filepath, metadata, and layer
    - scanDirectory on non-existent directory returns empty array (not throw)
  </behavior>
  <action>
    1. Install `yaml` package: `cd mcp-server && npm install yaml`

    2. Create `mcp-server/src/registry/types.ts` with these types:
       - `Layer = "L0" | "L1" | "L2"`
       - `PatternMetadata` interface with required fields: `scope: string[]`, `sources: string[]`, `targets: string[]` and optional fields: `version?: number`, `last_updated?: string`, `description?: string`, `slug?: string`, `status?: string` (default "active"), `excludable_sources?: string[]`
       - `RegistryEntry` interface: `slug: string`, `filepath: string`, `metadata: PatternMetadata`, `layer: Layer`, `project?: string`
       - `FrontmatterResult` interface: `data: Record<string, unknown>`, `content: string`

    3. Create `mcp-server/src/registry/frontmatter.ts`:
       - Export `parseFrontmatter(raw: string): FrontmatterResult | null`
       - Strip BOM (\uFEFF) at start
       - Check starts with `---\n` or `---\r\n`
       - Find closing `---` on its own line (handle both LF and CRLF)
       - Use `parse` from `yaml` package for the YAML portion
       - Return null on any error (invalid YAML, missing delimiters)
       - Import `parse as parseYaml` from `yaml` (ESM import)

    4. Create `mcp-server/src/registry/scanner.ts`:
       - Export `async function scanDirectory(dirPath: string, layer: Layer, project?: string): Promise<RegistryEntry[]>`
       - Read directory with `readdir` from `node:fs/promises`
       - Filter for `.md` files only
       - For each .md file, read content and call `parseFrontmatter`
       - Skip files without valid frontmatter (silently)
       - Derive slug from filename: remove `.md` extension. Special case: `enterprise-integration-proposal.md` maps to slug `enterprise-integration`
       - Validate metadata has `scope`, `sources`, `targets` arrays (skip files missing required fields)
       - Construct RegistryEntry with the slug, absolute filepath, parsed metadata, layer, and optional project
       - Return empty array if directory doesn't exist or readdir fails (log warning, don't throw)
       - Use `logger` from `../utils/logger.js` for warnings

    5. Write tests FIRST (RED), then implement (GREEN). Follow existing test patterns using vitest describe/it/expect.
  </action>

  </verify>
  <done>

    - types.ts exports PatternMetadata, RegistryEntry, Layer, FrontmatterResult
    - parseFrontmatter correctly parses valid YAML frontmatter and returns null for invalid input
    - scanDirectory discovers .md files with valid frontmatter and returns RegistryEntry[]
    - All unit tests pass
    - `cd mcp-server && npx tsc` compiles clean
