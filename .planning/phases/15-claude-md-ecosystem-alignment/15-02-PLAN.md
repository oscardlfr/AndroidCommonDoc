  <behavior>
    - Template structure: validates identity header (Layer, Inherits, Purpose blockquote), mandatory sections present, delegation statement present
    - Line count: warns if >150 lines, errors if >200 lines per file
    - Canonical coverage: loads canonical-rules.json, checks each rule at the assigned layer's CLAUDE.md is present (keyword/pattern match), reports missing coverage percentage
    - Circular reference detection: parses references/imports in each CLAUDE.md, errors if L0 references L1/L2 project names in generic rules (Developer Context section exempt), errors if L1 references L2
    - Override validation: parses "L0 Overrides" table, validates each override references a valid rule ID from canonical-rules.json, warns if overriding non-overridable rules
    - Version consistency: cross-checks version numbers found in CLAUDE.md files against versions-manifest.json, warns on mismatches
    - Cross-file duplicate detection: finds rules duplicated verbatim across layers, warns about drift risk
    - Multi-project support: when no project specified, discovers all projects via discoverProjects and validates each CLAUDE.md
    - Returns structured JSON matching ValidationResult pattern (valid, errors, warnings, issues, summary)
  </behavior>

**Test file** (`validate-claude-md.test.ts`):

Create test suites for each validation dimension:

1. `describe("template structure")` -- Test identity header detection, missing sections, delegation statement
2. `describe("line count and budget")` -- Test warning at >150, error at >200
3. `describe("canonical coverage")` -- Test rule matching against mock canonical-rules.json, coverage percentage
4. `describe("circular references")` -- Test L0->L1/L2 detection, L1->L2 detection, Developer Context exemption
5. `describe("override validation")` -- Test valid overrides pass, invalid rule IDs error, non-overridable warnings
6. `describe("version consistency")` -- Test version mismatch detection against mock versions-manifest.json
7. `describe("cross-file duplicates")` -- Test verbatim rule duplication detection across layers

Use in-memory CLAUDE.md content strings (not filesystem reads) for unit tests. Mock file reads for canonical-rules.json and versions-manifest.json.

**Implementation** (`validate-claude-md.ts`):

Follow the validate-skills.ts pattern exactly:
- Export `registerValidateClaudeMdTool(server: McpServer, limiter?: RateLimiter): void`
- Use zod schema for input: `{ project?: string, check_canonical?: boolean (default true), check_versions?: boolean (default true) }`
- Reuse `ValidationIssue` and `ValidationResult` interfaces (import from validate-skills.ts or define compatible ones)
- Use `getToolkitRoot()` for path resolution
- Use `discoverProjects()` for multi-project discovery when no project specified
- Load `canonical-rules.json` from `docs/guides/` at toolkit root
- Load `versions-manifest.json` from toolkit root
- Use `checkRateLimit()` guard
- Return MCP content response with JSON stringified result

Validation checks (in order):
1. **File discovery**: Find CLAUDE.md at project root (or ~/.claude/ for global)
2. **Template structure**: Parse identity header blockquote for Layer/Inherits/Purpose
3. **Line count**: Count lines, warn >150, error >200
4. **Section presence**: Check for expected sections per layer
5. **Canonical coverage** (if check_canonical): Load rules, match by keyword patterns in the correct layer's file
6. **Circular references**: Scan for project name references in L0 generic sections, scan for upward layer references
7. **Override validation**: Parse L0 Overrides table, validate IDs against canonical rules
8. **Version consistency** (if check_versions): Extract version patterns (e.g., "Koin 4.1.1", "Kotlin 2.3.10"), compare against versions-manifest.json
9. **Cross-file duplicates**: Compare rule text across layer files, flag verbatim matches

For canonical coverage matching, use keyword/pattern heuristics (not exact string match). Each canonical rule should have identifiable keywords that can be grepped in the CLAUDE.md content. Example: rule "UiState: ALWAYS sealed interface" matches if the CLAUDE.md contains "sealed interface" AND "UiState" in the same section.

Export internal validation functions for testability:
- `validateTemplateStructure(content: string, layer: string): ValidationIssue[]`
- `validateCanonicalCoverage(content: string, layer: string, rules: CanonicalRule[]): ValidationIssue[]`
- `detectCircularReferences(files: {path: string, layer: string, content: string}[]): ValidationIssue[]`
- `validateOverrides(content: string, rules: CanonicalRule[]): ValidationIssue[]`
- `checkVersionConsistency(content: string, manifest: Record<string, string>): ValidationIssue[]`
