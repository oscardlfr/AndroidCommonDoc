<task type="auto">
  <name>Task 2: Design CLAUDE.md template reference document</name>
  <files>docs/guides/claude-md-template.md</files>
  <action>
Create the standard CLAUDE.md template as an L0 pattern doc at `docs/guides/claude-md-template.md` with proper YAML frontmatter.

Frontmatter:
```yaml
---
scope: [claude-md, template, context-management]
sources: [anthropic-claude-code]
targets: [all]
slug: claude-md-template
status: active
layer: L0
category: guides
description: Standard template for CLAUDE.md files across the KMP ecosystem with L0/L1/L2 delegation
version: "1.0.0"
last_updated: "2026-03-16"
---
```

Template sections (in order):

**1. Identity Header (MANDATORY for all layers)**
Every CLAUDE.md starts with this block. Format:
```markdown
# {Project Name} Project Rules

> **Layer:** L{0|1|2} ({Generic KMP | Ecosystem Library | Application})
> **Inherits:** {inheritance chain}
> **Purpose:** {one-line description}

{Delegation statement}
{Scope statement}
```

Examples for each layer:
- L0 global: No inheritance, states it provides shared rules
- L0 toolkit: Inherits L0 global, adds toolkit-specific
- L1: Inherits L0, states module-specific scope
- L2: Inherits L0 + L1, states application-specific scope

**2. Critical Rules / Key Architecture Decisions**
Most important behavioral rules. For L0 this is architecture table, error handling, ViewModel rules. For L1/L2 this is project-specific decisions.

**3. Domain-Specific Section(s)**
Layer-dependent sections:
- L0 global: Source sets, naming, testing, DI, navigation, compose
- L0 toolkit: MCP server, skills/agents, pattern docs, development rules
- L1: Module catalog, API/-impl rules, how-to guides
- L2: Module structure, mandatory doc consultation, key dependencies

**4. Build Commands**
Essential build/test/run commands in fenced code block. Keep to 3-5 commands.

**5. L0 Overrides (OPTIONAL -- only if project overrides L0 defaults)**
Markdown table format:
```markdown
## L0 Overrides

| L0 Rule ID | Override | Reason |
|------------|----------|--------|
| DI-01 | Dagger/Hilt 2.52 | Corporate standard |
```
validate-claude-md checks that override IDs reference valid rules in canonical-rules.json.

**6. Temporal Context (OPTIONAL -- time-bounded project context)**
For active sprints, parallel tracks, release phases. Example: DawSync Wave 1 tracks. Clearly marked as temporal so it can be removed when no longer relevant.

**7. Team/Agent Rules (OPTIONAL -- only in L0 global)**
Git flow, worktree rules, agent file ownership. Lives in L0 since it applies globally.

Budget constraints documented:
- Each CLAUDE.md file: <150 lines
- Each project initial context load: <4000 tokens
- Developer context section: user-scoped, clearly marked for corporate replacement

Document the delegation model:
- L0 global auto-loads via Claude Code's native `~/.claude/CLAUDE.md` mechanism
- L1/L2 project CLAUDE.md files add project-specific context only
- No @import for cross-project references (known bugs per GitHub issue #8765)
- @import allowed only for local project files
- validate-claude-md enforces structure, direction, and consistency

Include anti-patterns section:
- Duplicating L0 rules in L1/L2
- L0 referencing downstream project names in generic rules
- Upward references (L1 -> L2, L0 -> L1)
- Bloated files (>150 lines)
- Version number duplication across layers

Keep the template doc under 200 lines (it's an L0 guide doc, not a hub).
